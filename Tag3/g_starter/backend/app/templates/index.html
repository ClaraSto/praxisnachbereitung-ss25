MQTT: Neues Gerät per Topic anlegen

Der Service hört auf das Topic device/new. Payload (JSON) z. B.:

{ "device_id": 1234, "device_name": "SuperSmartphone3000", "location_name": "Berlinhausen", "note": "This is optional" }

Der Service hört auf das Topic assinment/issue. Payload (JSON) z. B.:

{ "device_id: 1234, "personal_id": 2345, "personal_name": "Müller-Meier", "issued_at": "06.06.1999" }

Der Service hört auf das Topic assignment/return. Payload (JSON) z. B.:

{ "device_id": 1234, "returned_at": 12.12.2012, "grade_value": "1,7" }
